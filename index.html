What is inkboom

A design memory
for your agent

When an AI generates a UI, it starts from scratch every time — inventing colours, picking arbitrary spacing, using whatever font stack feels right in the moment. The result is inconsistent across pages and sessions.

inkboom fixes this. You define your palette, typography, spacing scale, and component library once. The agent reads it back as resolved values and writes CSS custom properties directly from your tokens.

Every project lives at ~/.inkboom/projects/<slug>/project.json. No database. No network. One file per project.

Quick start Token schema
project.json (abbreviated) 
{
  "slug": "my-saas",
  "tokens": {
    "color": {
      "brand": {
        "primary": {
          "type": "color",
          "value": "#4F46E5"
        }
      }
    },
    "icons": {
      "library": "lucide",
      "icons": ["lock", "zap", "globe"]
    }
  },
  "designSystem": {
    "components": [{
      "id": "feature-card",
      "props": {
        "icon": { "type": "icon" },
        "title": { "type": "string" }
      },
      "tokenBindings": {
        "iconColor": "{color.brand.primary}"
      }
    }]
  }
}
Workflow

Four calls. A complete design system.

01
create_project

Assign a URL-safe slug and a plain-English brief. Creates an empty project file at ~/.inkboom/projects/<slug>/project.json.

02
set_tokens

Provide the full token artifact — colours (6-digit hex, WCAG-validated), spacing, typography, shadows, radii, layout breakpoints, icon library. Validated against a Zod schema before writing.

03
set_design_system

Define all components with tokenBindings referencing token paths like {color.brand.primary}. Must include the 20 required base components.

04
get_design_system → generate HTML

Returns components with all bindings resolved to raw values. The agent writes CSS custom properties directly from these values — never from token references.

Features

Everything the agent needs
to stay consistent

Token storage

Validated token artifacts — colours, spacing, type, shadows, radii, icons. One file per project. Persistent across sessions.

Component library

20 required base components plus unlimited custom ones. Each component defines props, slots, and token bindings.

Resolved bindings

get_design_system walks all {token.path} references and returns raw values. The agent writes #4F46E5, never a reference.

Schema validation

Every write is validated by Zod. Hex must be 6-digit. Spacing must have units. Breakpoints must be strictly increasing. Bad tokens never land.

Icon props

Declare tokens.icons with a Lucide or Heroicons set. Components can have type: "icon" props. The agent inlines correct SVG paths.

Incremental updates

Use add_component and remove_component to patch the design system without rewriting everything. Iterate without blowing up your project.
