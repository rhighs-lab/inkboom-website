What it gives you

Everything the agent needs
to design consistently

Token storage

Save colours, spacing, typography, shadows, radii and icons as a validated token artifact. The agent reads them back verbatim.

Component library

Define 20+ components — button, card, heading, feature-card — with token bindings. The agent generates HTML using your exact system.

Resolved bindings

get_design_system returns raw values, not references. The agent writes #4F46E5, never {color.brand.primary}.

Patch and iterate

Update tokens, add or remove components, and regenerate pages. Every project is a persistent store the agent can refine over time.

Icon props

Declare tokens.icons with any Lucide or Heroicons set. Components get typed icon props; the agent inlines correct SVG paths.

Schema validation

Every set_tokens and set_design_system call validates against a strict JSON schema before saving. Bad tokens never land.

Workflow

Four calls.
A complete design system.

01
create_project

Assign a slug and a plain-English brief. inkboom creates an empty project and waits for your tokens.

02
set_tokens

Provide a full token artifact — colours, spacing, typography, shadows, radii, layout breakpoints, icon library. Validated on write.

03
set_design_system

Define all components with token bindings. Each binding is a reference like {color.brand.primary} resolved to a raw value on read.

04
get_design_system → generate

The agent reads resolved values and writes pixel-perfect, consistent HTML and CSS — with no hardcoded hex codes floating around the codebase.

SKILL.md usage example 
// 1. Create a project
create_project({
  slug: "my-site",
  brief: "Minimal SaaS landing. Light, indigo accent."
})

// 2. Set tokens
set_tokens({
  projectId: "my-site",
  tokens: {
    color: { brand: { primary: {
      type: "color", value: "#4F46E5"
    }}}}
})

// 3. Add a component
add_component({
  projectId: "my-site",
  component: {
    id: "feature-card",
    props: {
      icon: { type: "icon" },
      title: { type: "string" }
    },
    tokenBindings: {
      iconColor: "{color.brand.primary}"
    }
  }
})

// 4. Generate HTML using resolved values
get_design_system({ projectId: "my-site" })
// → iconColor: "#4F46E5"  (not a reference)

Design tokens

One source of truth.
Never drift.

inkboom validates every token against a strict JSON schema before persisting it. Hex must be 6-digit. Spacing must include units. Colours must pass WCAG AA contrast. The agent never writes a bad token.

Components declare {color.brand.primary} in their tokenBindings. get_design_system resolves every reference to a raw value — so generated HTML never contains a token reference, only the final computed value.
