Component schema

The design system is a JSON object containing an array of component definitions. It is passed as the designSystem argument to set_design_system.

Tokens must exist before a design system can be saved. Token binding references are stored as-is and resolved to raw values only when get_design_system is called.

DesignSystem

FieldTypeRequiredDescription
$schemastringyes"https://design-tokens.org/schema.json"
_meta.versionstringyesSemver string
_meta.generatedAtstringyesISO 8601 timestamp
componentsComponent[]yesArray of component definitions. Maximum 60 components.

Component

FieldTypeRequiredDescription
idstringyesKebab-case, starts with a letter. Pattern: /^[a-z][a-z0-9-]*$/. Must be unique within the design system.
displayNamestringyesHuman-readable name, e.g. "Feature Card"
categoryenumyesSee categories
descriptionstringyesAt least 1 character. Explains what the component is for.
isContainerbooleanyesWhether this component can hold other components
allowedChildren"*" | string[] | "none"yesWhich component IDs can be nested inside. Use "*" for any, "none" for leaf components.
allowedParents"*" | string[] | "root"yesWhich component IDs can contain this. Use "root" for top-level only.
propsRecord<string, Prop>yesComponent prop definitions. Can be empty {}.
slotsstring[]yesNamed content slots. Can be empty [].
tokenBindingsRecord<string, string | VariantBinding>yesMaps CSS-like property names to token paths or literal values.
requiredBindingKeysBindingKey[]noSpacing keys that must be present in tokenBindings. Defaults to [].

Props

Each key in props is a prop name mapping to a Prop definition:

FieldTypeDescription
typeenum"string" | "number" | "boolean" | "enum" | "color" | "size" | "slot" | "icon"
requiredbooleanWhether the prop must always be provided. Default false.
defaultstring | number | boolean | nullDefault value when not provided.
descriptionstringDocumentation string for the prop.
valuesstring[]Allowed values, only used when type === "enum".
contentContentDefOptional content kind hint: "text", "richText", "image", "link", "number", "date".

The icon prop type

Props with type: "icon" signal to the agent that this prop should receive an icon name from tokens.icons.icons[]. The agent is expected to inline the correct SVG path data for that icon — no CDN, no icon sprite.

"props": {
  "icon": {
    "type":        "icon",
    "required":    true,
    "description": "Lucide icon name from tokens.icons"
  },
  "variant": {
    "type":   "enum",
    "values": ["primary", "secondary", "ghost"],
    "default": "primary"
  }
}

Token bindings

Token bindings map visual properties to token paths. They are resolved by get_design_system.

Static binding

A string value. Either a token path in {dot.notation} or a literal CSS value:

"tokenBindings": {
  "backgroundColor": "{color.brand.primary}",  // → "#4F46E5"
  "padding":          "{spacing.lg}",           // → "24px"
  "borderWidth":      "1px"                    // literal pass-through
}

Variant binding

An object mapping enum prop values to token paths. Useful for components with size or variant props:

"tokenBindings": {
  "fontSize": {
    "sm": "{typography.fontSize.sm}",  // → "13px"
    "md": "{typography.fontSize.md}",  // → "15px"
    "lg": "{typography.fontSize.lg}"   // → "18px"
  }
}

requiredBindingKeys

Components that have spacing concerns should declare which spacing keys they require. The validator enforces these at set_design_system time. Allowed keys:

paddingpaddingXpaddingYpaddingToppaddingBottom paddingLeftpaddingRightgapmarginXmarginY marginTopmarginBottommarginLeftmarginRight

Required components

The design system must include these 20 component IDs. set_design_system will fail if any are missing:

buttoninputtextheadingcontainer cardimagedividerbadgelink checkboxselecttextareatableavatar alerttabsdialogprogressspinner

You can add any number of custom components on top of these via add_component after the initial set_design_system.

Full component example

feature-card component definition
{
  "id":           "feature-card",
  "displayName":  "Feature Card",
  "category":     "data-display",
  "description":  "Feature highlight with icon and title.",
  "isContainer":  false,
  "allowedChildren": "none",
  "allowedParents":  "*",
  "props": {
    "icon": {
      "type":        "icon",
      "required":    true,
      "description": "Lucide icon name from tokens.icons"
    },
    "title": {
      "type":     "string",
      "required": true,
      "content":  { "kind": "text" }
    },
    "description": {
      "type":    "string",
      "content": { "kind": "text" }
    }
  },
  "slots": [],
  "tokenBindings": {
    "backgroundColor": "{color.background.surface}",
    "borderColor":     "{color.border.default}",
    "borderRadius":    "{borderRadius.relaxed}",
    "padding":         "{spacing.lg}",
    "iconColor":       "{color.brand.primary}",
    "titleFontSize":   "{typography.fontSize.lg}",
    "titleFontWeight": {
      "value": "{typography.fontWeight.semibold}"
    }
  },
  "requiredBindingKeys": ["padding"]
}

// After get_design_system, tokenBindings are resolved:
{
  "backgroundColor": "#FAFAFA",
  "borderColor":     "#E4E4E7",
  "borderRadius":    "8px",
  "padding":         "24px",
  "iconColor":       "#4F46E5",
  "titleFontSize":   "18px",
  "titleFontWeight": { "value": "600" }
}
