MCP tools

inkboom exposes 9 tools via the MCP Streamable HTTP transport on POST http://localhost:4778/mcp. All tools return structured JSON. Errors include a code field and a human-readable message.

Call order: create_project → set_tokens → set_design_system. add_component and remove_component require an existing design system. get_tokens and get_design_system require tokens to have been set.
ToolDescription
create_projectCreate an empty project
delete_projectDelete a project permanently
get_tokensRead the token artifact
set_tokensWrite (and validate) the token artifact
get_design_systemRead components with resolved token bindings
set_design_systemWrite (and validate) all components
list_componentsList component IDs, names, categories
add_componentAppend a single component
remove_componentRemove a component by ID

create_project

Creates an empty project at ~/.inkboom/projects/<slug>/project.json. The project has no tokens and no design system until you call set_tokens.

Parameters

NameTypeRequiredDescription
slugstringyesURL-safe identifier. Lowercase alphanumeric + hyphens. Min 1 char.
briefstringyesPlain-English description of the design context. 10–2000 chars. Stored for reference.
overwritebooleannoReplace an existing project with the same slug. Defaults to false. Set to true to reset a project.

Response

{
  "projectId":  "my-site",
  "createdAt":  "2026-01-01T12:00:00.000Z",
  "message":    "Empty project created. Use set_tokens to add design tokens..."
}

delete_project

Deletes the project directory and all its contents. Irreversible.

Parameters

NameTypeRequiredDescription
projectIdstringyesThe slug of the project to delete.

Response

{ "success": true }

get_tokens

Returns the complete token artifact as stored. Token references are not resolved — you get the raw JSON exactly as it was saved by set_tokens.

Parameters

NameTypeRequired
projectIdstringyes

Response

Returns the full TokenValuesArtifact object. See Token schema.

set_tokens

Validates and persists the token artifact. This replaces any existing tokens for the project. The validation pipeline runs Zod schema checks plus business rules (required categories, strictly increasing breakpoints).

Parameters

NameTypeRequiredDescription
projectIdstringyesTarget project slug.
tokensTokenValuesArtifactyesComplete token object. See Token schema.

Response (success)

{ "success": true }

Response (validation failure)

{
  "error":  "VALIDATION_FAILED",
  "errors": [{
    "code":               "MISSING_REQUIRED_CATEGORY",
    "path":               "/shadow",
    "message":            "Required token category \"shadow\" is missing",
    "expectedConstraint": "\"shadow\" token category is required"
  }]
}

get_design_system

Returns all components with tokenBindings fully resolved to raw values. This is the tool the agent calls before generating HTML — all {token.path} references are walked and replaced with their final computed value.

If no design system has been set yet, returns { "designSystem": null }.

Parameters

NameTypeRequired
projectIdstringyes

Response

{
  "designSystem": {
    "components": [{
      "id": "feature-card",
      // ...
      "tokenBindings": {
        "iconColor":    "#4F46E5",  // was "{color.brand.primary}"
        "padding":      "24px",    // was "{spacing.lg}"
        "borderRadius": "8px"      // was "{borderRadius.relaxed}"
      }
    }]
  }
}

set_design_system

Validates and persists all components. Requires tokens to exist first. Replaces any existing design system.

Validation checks: all 20 required component IDs are present, requiredBindingKeys are present in each component's tokenBindings, component IDs match the pattern /^[a-z][a-z0-9-]*$/.

Parameters

NameTypeRequiredDescription
projectIdstringyesTarget project slug.
designSystemDesignSystemyesComplete design system object. See Component schema.

Error: tokens not set

{
  "error":   "NO_TOKENS",
  "message": "Project has no design tokens. Use set_tokens first."
}

list_components

Returns a lightweight list of all components in the design system — ID, displayName, category, and description only. Useful for checking what exists without fetching resolved bindings.

Parameters

NameTypeRequired
projectIdstringyes

Response

{
  "components": [
    { "id": "button",       "displayName": "Button",       "category": "action"       },
    { "id": "feature-card", "displayName": "Feature Card", "category": "data-display" }
  ]
}

add_component

Appends a single component to the existing design system. Fails if a component with the same ID already exists. Use this for incremental updates instead of rewriting the whole design system via set_design_system.

Parameters

NameTypeRequiredDescription
projectIdstringyesTarget project slug.
componentComponentyesThe component to add. See Component.

Response (success)

{ "success": true, "componentId": "pricing-card" }

Error: component already exists

{
  "error":   "COMPONENT_EXISTS",
  "message": "Component \"pricing-card\" already exists."
}

remove_component

Removes a component from the design system by ID. Fails if the component does not exist.

Parameters

NameTypeRequiredDescription
projectIdstringyesTarget project slug.
componentIdstringyesID of the component to remove.

Error: not found

{
  "error":   "COMPONENT_NOT_FOUND",
  "message": "Component \"old-card\" not found."
}

Error codes

CodeReturned byMeaning
VALIDATION_FAILEDset_tokens, set_design_system, add_componentSchema or business validation failed. The errors array contains details.
NO_TOKENSset_design_system, add_componentTokens must be set before a design system can be written.
NO_DESIGN_SYSTEMadd_component, remove_componentA design system must exist before components can be added or removed.
COMPONENT_EXISTSadd_componentA component with this ID already exists. Use set_design_system to overwrite, or remove_component first.
COMPONENT_NOT_FOUNDremove_componentNo component with the given ID exists in the design system.
PROJECT_NOT_FOUNDany tool with projectIdNo project with the given slug was found on disk.
PROJECT_EXISTScreate_projectA project with this slug already exists. Pass overwrite: true to reset it.
MISSING_REQUIRED_BINDING_KEYset_design_system, add_componentA component declared requiredBindingKeys but one or more keys are absent from tokenBindings.

Debug mode

Set the environment variable INKBOOM_DEBUG=1 before starting the server to log raw token/component payloads on validation failure. Helpful for diagnosing why a large token artifact is rejected.

❯ INKBOOM_DEBUG=1 pnpm mcp
