Token schema

The token artifact is a single JSON object validated by a Zod schema before being persisted. It is passed as the tokens argument to set_tokens.

All five top-level categories — color, spacing, typography, shadow, borderRadius — are required. layout and icons are optional.

Top-level structure

FieldTypeRequiredDescription
$schemastringyesMust be "https://design-tokens.org/schema.json"
_meta.versionstringyesSemver string, e.g. "1.0.0"
_meta.generatedAtstringyesISO 8601 timestamp
colorColorGroupyesBrand, semantic, text, background, border colours
spacingSpacingGroupyesxs → 2xl dimension scale (3xl optional)
typographyTypographyGroupyesfontFamily, fontSize, fontWeight, lineHeight
shadowShadowGroupyessubtle, prominent (focus optional)
borderRadiusBorderRadiusGroupyestight, relaxed (full optional)
layoutLayoutGroupnocontainerMaxWidth plus sm/md/lg/xl breakpoints
iconsIconsGroupnoIcon library declaration with icon name list

color

All colour values must be 6-digit lowercase hex (#rrggbb). 3-digit shorthand is rejected.

PathRequiredDescription
color.brand.primaryyesMain brand colour — used for CTAs, accent elements
color.brand.secondaryyesSupporting brand colour
color.semantic.erroryesError / destructive state
color.semantic.warningyesWarning state
color.semantic.successyesSuccess / positive state
color.semantic.neutralyesNeutral / informational
color.text.*yesOpen record — define as many text colours as needed (e.g. primary, secondary, inverse)
color.background.*yesOpen record — page, surface, subtle, dark, etc.
color.border.*yesOpen record — default, strong, dark, etc.
WCAG AA contrast: text colours must achieve ≥4.5:1 contrast ratio against their intended backgrounds. The SKILL.md reminds the agent to check this before writing tokens. 
"color": {
  "brand": {
    "primary":   { "type": "color", "value": "#4F46E5" },
    "secondary": { "type": "color", "value": "#000000" }
  },
  "semantic": {
    "error":   { "type": "color", "value": "#DC2626" },
    "warning": { "type": "color", "value": "#D97706" },
    "success": { "type": "color", "value": "#16A34A" },
    "neutral": { "type": "color", "value": "#6B7280" }
  },
  "text": {
    "primary":   { "type": "color", "value": "#000000" },
    "secondary": { "type": "color", "value": "#6B7280" },
    "inverse":   { "type": "color", "value": "#FFFFFF" }
  },
  "background": {
    "base":    { "type": "color", "value": "#FFFFFF" },
    "surface": { "type": "color", "value": "#FAFAFA" },
    "subtle":  { "type": "color", "value": "#F4F4F5" }
  },
  "border": {
    "default": { "type": "color", "value": "#E4E4E7" },
    "strong":  { "type": "color", "value": "#A1A1AA" }
  }
}

spacing

All spacing values must include a CSS unit (px, rem, or em). Values must be strictly increasing: xs < sm < md < lg < xl < 2xl. A geometric ratio of ~1.5× between steps is recommended.

KeyRequiredExample
spacing.xsyes"4px"
spacing.smyes"8px"
spacing.mdyes"16px"
spacing.lgyes"24px"
spacing.xlyes"40px"
spacing.2xlyes"64px"
spacing.3xlno"96px"

typography

PathRequiredNotes
typography.fontFamily.bodyyesFull font stack string
typography.fontFamily.headingyesFull font stack string
typography.fontSize.xs – 2xlyesMust have unit. 3xl is optional.
typography.fontWeight.*yesOpen record. Values 100–900 (int) or "700" (3-digit string).
typography.lineHeight.*yesOpen record. Unitless numbers like 1.5.

shadow

Shadow values can be a raw CSS string or a structured object:

// String form
"subtle": {
  "type": "shadow",
  "value": "0 1px 3px 0 rgba(0,0,0,.10)"
}

// Structured object form
"prominent": {
  "type": "shadow",
  "value": {
    "offsetX": "0px",
    "offsetY": "4px",
    "blur":    "12px",
    "spread":  "0px",
    "color":   "rgba(0,0,0,.12)"
  }
}

borderRadius

KeyRequiredExample
borderRadius.tightyes"2px" or "4px"
borderRadius.relaxedyes"8px"
borderRadius.fullno"9999px" for pill shapes

layout (optional)

Declare container max-width and responsive breakpoints. Breakpoints must be strictly increasing.

"layout": {
  "containerMaxWidth": { "type": "dimension", "value": "1120px" },
  "sm": { "type": "dimension", "value": "640px"  },
  "md": { "type": "dimension", "value": "768px"  },
  "lg": { "type": "dimension", "value": "1024px" },
  "xl": { "type": "dimension", "value": "1280px" }
}

If layout is omitted, the SKILL.md tells the agent to default to sm: 640px / md: 768px / lg: 1024px / xl: 1280px.

icons (optional)

Declares the icon library the agent should use when rendering components with type: "icon" props. The agent inlines SVG paths — no CDN, no icon fonts.

FieldRequiredDescription
libraryyesLibrary name, e.g. "lucide" or "heroicons"
styleno"outline" | "solid" | "mini"
versionnoPinned version string
iconsyesArray of icon names to use. The agent references these when populating icon props.
 
"icons": {
  "library": "lucide",
  "style":   "outline",
  "icons": [
    "lock", "zap", "globe",
    "cpu",  "git-branch", "terminal"
  ]
}
